# {{site.data.keyword.vra_full}} の概要
{: #getting-started}

{{site.data.keyword.vra_full}} (VRA) は、x86 ベアメタル・サーバーに対応した最新の Vyatta 5600 オペレーティング・システムを提供します。 これは、高可用性 (HA) 構成またはスタンドアロン構成として提供され、ファイアウォール、トラフィック・シェーピング、ポリシー・ベースの経路指定、 VPN、および他のフィーチャーを備えたフル装備のエンタープライズ・ルーターを介して、プライベートおよびパブリックのネットワーク・トラフィックを選択的に経路指定できるようにします。

VRA 最小サーバー条件は、ネットワーク容量 10 Gbps につき 8 GB の RAM および 1 CPU コアです。 例えば、デュアル 10 Gbps のパブリックおよびプライベートのアップリンクを持つシステムには、少なくとも 4 つのコアが必要です。 また、暗号化を使用して VPN サービスをセットアップすることを意図している場合は、さらにコアを追加することが必要な場合があります。 VPN サービス用のコアを追加すると、データの経路指定と同時暗号化/暗号化解除の際に、VRA が重い負荷によって動きが取れないということはなくなります。

## {{site.data.keyword.vra_full}} の注文
{: #order-vra}

VRA を注文するには、以下の手順を実行します。

1. ブラウザーで [IBM Cloud UI コンソール![外部リンク・アイコン](../../icons/launch-glyph.svg "外部リンク・アイコン")](https://cloud.ibm.com/gen1/infrastructure/provision/gateway){: new_window}の「ゲートウェイ・アプライアンス」ページを開き、アカウントにログインします。 

  このページには、[IBM Cloud カタログ![外部リンク・アイコン](../../icons/launch-glyph.svg "外部リンク・アイコン")](https://cloud.ibm.com)の左上にあるナビゲーション・メニューを選択し、**「クラシック・インフラストラクチャー」>「ネットワーク」>「ゲートウェイ・アプライアンス」**を選択してアクセスすることもできます。
  {: tip}

2. **「ゲートウェイ・ベンダー」**セクションで、**「AT&T」**オプションを選択します (これを選択すると、青いチェック・マークがボタンに表示されます)。その同じボタンのドロップダウンから、帯域幅 (20 Gbps または 2 Gbps) を選択します。

  	<img src="images/ordering_vra.png" alt="図" style="width: 500px;"/>

3. **「ゲートウェイ・アプライアンス」**セクションで、**「ホスト名」**と**「ドメイン」**の名前情報を入力します。これらのフィールドには既にデフォルト情報が取り込まれているため、値が正しいことを確認してください。 必要に応じて**「高可用性」**オプションにチェック・マークを付け、希望するデータ・センターの**「場所」**と必要な特定の**「ポッド」**をドロップダウン・メニューから選択します。

  VLAN が既に関連付けられているポッドのみここに表示されます。リストされていないポッドにゲートウェイ・アプライアンスをプロビジョンしたい場合は、最初にそのポッドに VLAN を作成します。
　{: note}

4. **「構成」**セクションで、RAM および SSH 鍵を選択してプロセッサーを選択します (必要な場合)。

  <img src="images/ordering_vra_2.png" alt="図" style="width: 600px;"/>
  
  ステップ 2 で選択したライセンス・バージョンに基づいて、適切なプロセッサーが選択されます。ただし、異なる RAM 構成を選択することができます。
  {: note}

5. **「ストレージ・ディスク」** セクションで、ストレージ要件を満たすオプションを選択します。 

  RAID0 オプションと RAID1 オプションを使用すれば、ホット・スペア (1 次コンポーネントで障害が発生した場合に即時に稼働させることができるバックアップ・コンポーネント) と同様に、データ損失に対する保護を強化することができます。
  {: note}

  1 つの VRA につき最大 4 つのディスクを保持できます。RAID 構成はミラーリングされるため、1 つの RAID 構成の「ディスク・サイズ」が使用可能なディスク・サイズです。
  {: note}

  詳細なログを生成するネットワーク診断の実行を予定している場合は、デフォルトのディスク設定よりも大きな容量を予約してください。
  {: tip}

6. **「ネットワーク・インターフェース」**セクションで、**「アップリンク・ポート速度」**を選択します。デフォルトでは単一インターフェースが選択されていますが、冗長オプションと専用オプションもあります。要件に最適なものを選択してください。

  ネットワーク・インターフェースの**「アドオン (Add Ons)」**セクションでは、必要に応じて IPv6 アドレスを選択できます。また、組み込みのその他のデフォルト・オプションが表示されます。 
  
8. 選択内容を確認し、サード・パーティー・サービス契約を読んだことを確認してから、**「作成」**をクリックします。注文は自動的に検証されます。

注文が承認されると、{{site.data.keyword.vra_full}} のプロビジョニングが自動的に開始されます。 プロビジョニング・プロセスが完了すると、新しい VRA が「ゲートウェイ・アプライアンス」リスト・ページに表示されます。 ゲートウェイ名をクリックして「ゲートウェイの詳細」ページを開きます。 デバイスの IP アドレス、ログイン・ユーザー名、およびパスワードが表示されます。  

  <img src="images/gateway_details.png" alt="図" style="width: 500px;"/>

IBM Cloud カタログから VRA を注文して構成すると、デバイス自体も同じ設定で構成する必要があることに注意してください。
{: tip}

## VLAN およびゲートウェイ・アプライアンスの役割
{: #vlans-and-the-gateway-appliance-s-role}

VLAN (仮想 LAN) は、1 つの物理ネットワークを多数の仮想セグメントに分離するメカニズムです。 利便性のため、選択した複数の VLAN からのトラフィックは、単一のネットワーク・ケーブルで送信できます。このプロセスは一般的に「トランキング」と呼ばれます。

提供される {{site.data.keyword.vra_full}} は 2 つの部分に分けられます。VRA サーバーとゲートウェイ・アプライアンス・フィクスチャーです。 ゲートウェイ・アプライアンスは、ご使用の VRA と関連付けたい VLAN を選択するためのインターフェース (GUI と API) を提供します。 VLAN をゲートウェイ・アプライアンスと関連付けると、VLAN とそのすべてのサブネットがご使用の VRA に経路変更 (トランク) されるため、フィルタリング、転送、および保護を制御することが可能になります。 ゲートウェイ・アプライアンスに関連付けられているすべての VLAN は、VRA の接続先のスイッチ・ポートで許可され、その VLAN 上のすべてのサブネットは、VRA のパブリック VRRP IP (サブネットがパブリック・サブネットの場合) または VRA のプライベート VRRP IP (サブネットがプライベート・サブネットの場合) に静的に経路指定されます。このルーティングは、VRA がその背後にあるルーター、つまり、パブリック・トラフィックの場合はフロントエンド・カスタマー・ルーター (FCR)、プライベート・トラフィックの場合はバックエンド・カスタマー・ルーター (BCR) で行われます。 

VRRP はデフォルトで無効になっており、VLAN トラフィックが機能するためには VRRP を有効にする必要があることに注意してください。このことは、スタンドアロンの vyatta でも同様です。これは、関連付けられた VLAN 上のサブネットが、VRA に割り当てられた VRRP IP または仮想アドレスにルーティングされるためです。詳しくは、[VRRP 仮想 IP (VIP) アドレス](/docs/infrastructure/virtual-router-appliance?topic=virtual-router-appliance-working-with-high-availability-and-vrrp#vrrp-virtual-ip-vip-addresses)を参照してください。
{: important}

関連付けられた VLAN 内のサーバーは、ご使用の {{site.data.keyword.vra_full}} を介してのみ、他の VLAN から到達することが可能になります。つまり、VLAN をバイパスするか関連付けを解除しない限り、{{site.data.keyword.vra_full}} を回避することはできません。

デフォルトでは、新規のゲートウェイ・アプライアンスは、リムーバブルではない 2 つの「トランジット」VLAN (パブリックとプライベート 1 つずつ) に関連付けられます。 これらは通常、管理に使用され、VRA コマンドによって個別に保護できます。

Transit VLAN はファイアウォールやロード・バランサーのようなネットワーク・デバイス向けで、サーバーやコンテナーのような他のデバイスをインターネットから隔離したまま、トラフィックを経路指定することができます。

これと比べて「gateway」VLAN は、サーバーやコンテナーのようなデバイスがホストされている場所です。

VRA は、ゲートウェイ・アプライアンスを介して関連付けられた VLAN のみを管理できます。

「ゲートウェイ・アプライアンス詳細 (Gateway Appliances Details)」画面から VLAN を管理する方法については、「[VLAN の管理](/docs/infrastructure/virtual-router-appliance?topic=virtual-router-appliance-managing-your-vlans)」を参照してください。
